 
Project code: P01

Name : Smart Credit Policy Hub (UW.)

Detail : ใช้ประโยชน์จาก AI ในการเข้าถึงและประมวล ข้อมูล นโยบายเครดิต การสื่อความ และ ประกาศ Product ต่างๆ ที่อยู่ใน Sharepoint ของธนาคาร เพื่อช่วยเพิ่มประสิทธิภาพในการทำงาน โดย ลดปัญหาในการสืบค้นข้อมูล การตีความเนื้อหา 

Problem : search engine ของ Microsoft มีข้อจำกัดในการสืบค้นข้อมุลที่เก็บอยู่ใน SharePoint ของธนาคาร ซึ่งมีการเก็บอยู่ตามแต่ละหน่วยงาน และ หลากหลาย ดังนั้น ในการใช้งานจึงมักจะอิงจากประสบการณ์ของผู้ที่ทำอยู่เป็นประจำ ว่าอยู่หมวดใหน ตรงใหน Link ใหน หัวข้อย่อยใหน จนทำให้ต้องมีการสรุปเก็บข้อมูลที่แต่ละส่วนใช้เก็บไว้ในพื้นที่ตัวเอง อีกทั้งในบางครั้งต้องมีการเชื่อมโยงกับหมวดอื่นหมู่อื่น ซึ่งจะมีลักษณะคล้ายกับการอ่านกม. ที่ต้องเปิดทั้ง มาตรา และ คำพิพากษาศาลฏีกา ประกอบไปด้วยกัน ซึ่งเป็น Pain สำหรับหน่วยงานต่างๆที่ไม่คุ้นเคย หรือ สำหรับพนักงานที่ยังมีประสบการณ์ไม่มากในการทำความเข้าใจและปฏิบัติให้ได้ตามที่ธนาคารได้กำหนดแนวทางไว้

Solution : AI จะมีส่วนช่วยในการเข้าถึงข้อมูลซึ่งถูกเก็บไว้ใน SharePoint แล้วใช้ความสามารถด้าน LLM ในการประมวลเพื่อหาคำตอบที่ต้องการ

Outcome : เพิ่มประสิทธิภาพและความถูกต้องในการเข้าถึงข้อมูลซึ่งแต่ละหัวข้อหรือแต่ละเรื่อง สามารถลดเวลาลงได้อย่างน้อย 10 นาทีต่อหนึ่งหัวข้อ ซึ่งวันหนึ่งๆสามารถลดเวลาลงได้ 30นาที ซึ่งภายใต้เวลาที่จำกัดสามารถเอาเวลาที่ประหยัดได้ไปวิเคราะห์เนื้อหาอื่นๆที่มีความซับซ้อนมากขึ้นได้

Project code: P02

Name : AI-Assisted AML Alert Management (CO.)

Detail : ปัจจุบัน การแจ้งเตือนเกี่ยวกับการป้องกันการฟอกเงิน (AML) เพิ่มขึ้นอย่างมาก เนื่องจากการใช้บัญชี Nominee Accounts ในกิจกรรมที่ผิดกฎหมายที่เพิ่มขึ้น ซึ่งส่งผลให้เกิดปริมาณงานที่มากเกินกว่าที่ผู้ตรวจสอบจะจัดการได้ ดังนั้น โครงการนี้จึงมีเป้าหมายที่จะใช้ประโยชน์จาก AI และ RPA เพื่อจัดการกับจำนวนการแจ้งเตือนการฟอกเงินที่เพิ่มขึ้นอย่างมีประสิทธิภาพ และสนับสนุนการจัดการการแจ้งเตือนอย่างมีประสิทธิผล

Problem : ปริมาณงานคาดว่าจะเพิ่มขึ้นจนไม่สามารถจัดการได้ เนื่องจากการเพิ่มขึ้นของบัญชี Nominee Accounts ซึ่งส่งผลให้จำนวนการแจ้งเตือนเพิ่มขึ้นอย่างมาก


Solution : ใช้ Machine Learning ในการจัดประเภทการแจ้งเตือนตามระดับความรุนแรง โดยวิเคราะห์ข้อมูลจากโปรไฟล์ ธุรกรรม และกฎเกณฑ์ต่างๆ และใช้ RPA ในการสร้าง report โดยอัตโนมัติ

Outcome : Target to reduce 23% of investigators' workload.

Project code: P03

Name : Productivity & Portfolio Diversification (CM.)

Detail : ใช้ AI เพื่อเพิ่มผลการติดตามพอร์ตให้รวดเร็วมากขึ้น และการวิเคราะห์กระจายความเสี่ยงของพอร์ต (Portfolio Diversification Analysis) 

Problem : ธนาคารมีข้อมูลจำนวนมากและกระจัดกระจายจากหลายแหล่ง ทำให้การวิเคราะห์หลายมิติและการสร้าง dashboard visual ใช้เวลานาน จนไม่สามารถตอบคำถามจากผู้บริหารได้อย่างรวดเร็ว และยังไม่สามารถแยกความเสี่ยงของลูกค้าคุณภาพดีและไม่ดีได้อย่างชัดเจน.


Solution : การใช้ AI ในการวิเคราะห์และจำแนกลูกค้าที่มีคุณภาพดีและไม่ดีจากข้อมูลย้อนหลัง พร้อมทั้งแนะนำแนวทางนโยบายคุณภาพลูกค้าเพื่ออนุมัติคุณภาพลูกค้า ช่วยลดระยะเวลาในการวิเคราะห์และการสร้าง dashboard รวมถึงสรุปข้อมูลสำคัญให้รวดเร็วและมีประสิทธิภาพมากขึ้น

Outcome : การใช้ AI ในการวิเคราะห์และจำแนกลูกค้าที่มีคุณภาพดีและไม่ดีจากข้อมูลย้อนหลังช่วยให้สามารถประเมินคุณภาพลูกค้าได้อย่างแม่นยำและรวดเร็ว โดย AI สามารถแนะนำแนวทางนโยบายคุณภาพลูกค้าเพื่อการอนุมัติได้อย่างมีประสิทธิภาพ นอกจากนี้ยังช่วยลดระยะเวลาในการวิเคราะห์ข้อมูลและลดเวลาที่ใช้ในการสร้าง dashboard พร้อมทั้งสามารถสรุปข้อมูลสำคัญได้อย่างรวดเร็ว ทำให้การตัดสินใจในธุรกิจมีความทันสมัยและมีประสิทธิภาพมากขึ้น

Project code: P04

Name : Kxception (OR.)

Detail : พัฒนา AI ให้เป็น Partner คอยให้คำปรึกษาและเเนะนำเเนวทางที่คาดว่าจะทำให้ได้รับการอนุมัติ เมื่อเวลา User ต้องการขอ Exception กระบวนการต่างๆขอธนาคาร โดยเมื่อหารือจน AI เห็นว่าน่าจะได้รับการอนุมัติ AI จะ Auto เปิด Memo Workflow เพื่อขออนุมัติให้กับ User ตามลำดับขั้น

และ AI จะต้องสามารถทำสรุป Report การขอ Exception ตลอดจนพยากรณ์แนวโน้มการขอ Exception รวมทั้งให้ข้อเสนอเเนะต่างๆ ที่จะเป็นประโยชน์สำหรับการวิเคราะห์ข้อมูลและปรับปรุงกระบวนการ

Problem : เนื่องจากในช่วง 2-3 ปีที่ผ่านมา ธนาคารมีปริมาณ Case Exception เพิ่มสูงขึ้นอย่างต่อเนื่อง รวมทั้ง กระบวนการทดแทนที่นำมาเสนอขออนุมัติก็มีความหลากหลาย ไม่เป็นมาตรฐานตามคู่มือปฏิบัติงาน ทำให้ฝ่ายงานที่เกี่ยวข้อง ต้องมาพิจารณารายละเอียดเคสโดยละเอียด ซึ่งเป็นการเสียเวลา และ เพิ่มระยะเวลาในการพิจารณาสูงขึ้น ส่งผลให้ลูกค้าที่มาติดต่อขอ Exception มีแนวโน้มที่จะร้องเรียนทางการเพิ่มมากขึ้น

Solution : ช่วยใน 3 ส่วน คือ

1. ช่วยให้คำปรึกษาและแนะนำแนวทางที่คาดว่าจะช่วยให้การขออนุมัติ Exception มีโอกาสได้รับการอนุมัติมากขึ้น โดยการประมวลผลข้อมูลและพิจารณากระบวนการที่เกี่ยวข้องในธนาคาร
2. ช่วยลดเวลาการเปิด Memo Workflow เพื่อให้การขออนุมัติ Exception เป็นไปอย่างรวดเร็วและมีความแม่นยำสูง โดย AI จะช่วยในการตัดสินใจและเปิดการดำเนินการอัตโนมัติเมื่อเห็นว่าอนุมัติได้
3. ช่วยจัดทำสรุปและพยากรณ์แนวโน้มการขอ Exception เพื่อการวิเคราะห์และปรับปรุงกระบวนการในอนาคต ตลอดจนแนะนำกระบวนการ Exception ที่ควรยกระดับให้เป็นกระบวนการมาตรฐานของธนาคาร

Outcome : ตัวชี้วัด วัดได้จากหลายมิติ เช่น

- SLA ที่ลดลงการขออนุมัติ
- ลดชั่วโมงการทำงานและจำนวนบุคลากรที่ต้องใช้ในการหารือกระบวนการ Exception (ลด Man day โดยรวม)
- จำนวนเคสที่ลดลงในการขอหารือหรือส่งขออนุมัติ
- จำนวนกระบวนการมาตรฐานที่ยอมรับได้ มีเพิ่มสูงขึ้น

Project code: P05

Name : PonyKix (OR.)

Detail : พัฒนาโมเดล AI สำหรับช่วย Detect พฤติกรรมหรือคุณลักษณะของลูกค้าที่คาดว่าจะเป็นบัญชีม้า โดยฝึกสอน AI ด้วยข้อมูลลูกค้าที่เป็นบัญชีม้าที่ธนาคารได้รับแจ้งจากหน่วยงานทางการ (HR03, CFR) แล้วว่าเป็นบัญชีม้า โดยใช้เทคนิคของ Machine Learning และ Deep Learning ในการเรียนรู้จากข้อมูลที่มีอยู่และหาความสัมพันธ์ระหว่างตัวแปรต่างๆ ของข้อมูลลูกค้า ตลอดจนทดสอบและปรับปรุงโมเดลให้มีความแม่นยำเพียงพอสำหรับให้ฝ่ายงานที่เกี่ยวข้องใช้ในการชี้เป้าสำหรับการตรวจสอบรายการ

Problem : จากปริมาณ Nominee Account ที่เพิ่มสูงขึ้นอย่างต่อเนื่อง ธนาคารต้องแบกรับ Operation Cost ในการปิดบัญชีตามคำสั่งจากหน่วยงานทางการ (HR03, CFR) และต้องใช้เวลาทรัพยากรในการดำเนินการที่ไม่เกิดประโยชน์กับธนาคาร ดังนั้น โครงการนี้จึงมีเป้าหมายในการพัฒนา AI เพื่อช่วย ตรวจจับ Nominee Account โดยอัตโนมัติ เพื่อลดภาระงานที่เกิดจากคำสั่งดังกล่าว ซึ่งจะช่วยลดการใช้ทรัพยากรและลดจำนวนบัญชี Nominee ที่มีอยู่ในธนาคาร

Solution : การนำ AI และ Data Analytics มาใช้เป็น solution สำหรับการตรวจจับ Nominee Account ช่วยลดภาระงานที่เกี่ยวข้องกับการปิดบัญชีตามคำสั่งจากหน่วยงานทางการ และลดการใช้ทรัพยากรที่ไม่เกิดประโยชน์ โดยการใช้ Machine Learning ในการวิเคราะห์ข้อมูลลูกค้าและตรวจจับบัญชีที่มีความเสี่ยงสูงที่จะเป็นบัญชีม้า โดยสามารถทำได้ดังนี้:

1. รวบรวมและเตรียมข้อมูล: ใช้ข้อมูลลูกค้าที่มีการติดป้ายกำกับว่าเป็นบัญชีม้าหรือไม่ และข้อมูลการใช้งาน
2. การวิเคราะห์ด้วย Data Analytics: วิเคราะห์ข้อมูลเพื่อหาความสัมพันธ์และแนวโน้มที่บ่งชี้ถึงบัญชีม้า
3. การใช้ AI: ใช้เทคนิค Supervised Learning หรือ Unsupervised Learning เพื่อฝึกโมเดลในการตรวจจับบัญชีที่มีความเสี่ยง
4. การนำไปใช้จริง: นำโมเดลไปใช้ตรวจจับบัญชีใหม่ และลดภาระการตรวจสอบ และ เพิ่มความแม่นยำในการตรวจสอบ

การใช้ AI และ Data Analytics จะช่วยให้ธนาคารลดค่าใช้จ่ายในการปิดบัญชีม้า และเพิ่มประสิทธิภาพในการตรวจจับได้เร็วและแม่นยำขึ้น

Outcome : 1. ลดจำนวนบัญชีม้า
ตัวชี้วัด: จำนวนบัญชีม้าที่ถูกตรวจจับได้
ผลลัพธ์: ลดจำนวนบัญชีม้าที่ธนาคารต้องจัดการและลดคำสั่งจากหน่วยงานทางการ

2. เพิ่มความแม่นยำในการตรวจจับ
ตัวชี้วัด: Precision และ Recall
ผลลัพธ์: ตรวจจับบัญชีม้าได้แม่นยำขึ้น ลดการแจ้งเตือนผิดพลาด

3. ลดเวลาและทรัพยากรที่ใช้ในการดำเนินการ
ตัวชี้วัด: เวลาในการตรวจจับและปิดบัญชี
ผลลัพธ์: ลดเวลาในการตรวจสอบและลดภาระงานของพนักงาน

4. ประหยัดต้นทุน
ตัวชี้วัด: ค่าใช้จ่ายในการตรวจจับและปิดบัญชีม้า
ผลลัพธ์: ลดต้นทุนที่เกี่ยวข้องกับการจัดการบัญชีม้า

5. เพิ่มประสิทธิภาพการตรวจสอบ พิสูจน์ทราบ
ตัวชี้วัด: ความเร็วในการตรวจสอบ พิสูจน์ทราบ
ผลลัพธ์: ช่วยให้การตรวจสอบ พิสูจน์ทราบเร็วขึ้น และ แม่นยำมากขึ้น

Project code: P06

Name : Bank Statement and Social Security Documents Data Extraction (ZZ.)

Detail : Generative AI-Powered OCR for Financial and Social Security Documents in Vietnam

Problem : ข้อมูลในเอกสาร Bank Statement และ Social Security ไม่ได้ถูกจัดเก็บอย่างครบถ้วนสมบูรณ์เนื่องจาก การใช้ทีมปฏิบัติการในการจัดเจ็บมีระยะเวลาที่ต้องใช้ในการจัดเก็บพอสมควร

Solution : Generative AI ปัจจุบันเช่น ChatGPT model O1 นั้นสามารถทำ OCR ได้อย่างมีประสิทธิภาพ การใช้ Generative AI เข้ามาเป็นส่วนนึงในกระบวนการจะช่วยให้เราสามารถรวบรวมข้อมูลได้ครบถ้วนสมบูรณ์ ทั้งยังสามารถลดระยะเวลาในการทำงานของ Operation Team ลงได้

Outcome : - รวบรวมข้อมูลที่มีประโยชน์จากเอกสาร Bank Statement และ Social Security ได้ครบสมบูรณ์
- ลดระยะเวลาการทำงานของ Operation Team

Name : AI for Financial & ESG Data Accuracy and Risk Insights (CM., IY.)

Detail : การทำงานด้าน ความเสี่ยงเครดิต (credit risk) และ ความเสี่ยงทางสิ่งแวดล้อมและสังคม (ESG risk) ต้องอาศัยข้อมูลที่มีคุณภาพและเชื่อถือได้เป็นหลัก เพื่อให้การประเมินความเสี่ยงและการตัดสินใจทางธุรกิจเกิดขึ้นได้อย่างมีประสิทธิภาพ ดังนั้น การเก็บข้อมูลให้มีความแม่นยำและรวดเร็วจึงเป็นสิ่งสำคัญที่สุด ข้อมูลที่ดีจะช่วยให้การประเมินด้านความเสี่ยงสามารถทำได้อย่างถูกต้องและโปร่งใส พร้อมทั้งช่วยเพิ่มความมั่นใจในการวิเคราะห์และคาดการณ์ผลกระทบที่จะเกิดขึ้นในอนาคต โครงการที่ทางทีมสนใจประกอบไปด้วย 2 Phase ได้แก่

Phase 1. AI for Data Validation
1.1 กำหนดข้อมูล และ format ที่ต้องการจาก AI ให้ชัดเจน เพื่อให้ AI ทำการดึงข้อมูล และรายงานผลข้อมูล ในรูปแบบที่ User กำหนด เพื่อลดระยะเวลาทำงานของ User และพร้อมนำไปใช้งานต่อได้อย่างมีประสิทภาพ ตัวอย่างข้อมูลที่สำคัญ ได้แก่ Key Financial items, GHG Emission Scope, กำลังการผลิตของลูกค้า เป็นต้น 
Phase 2. AI Generating
2.1 พัฒนาข้อมูลสมบูรณ์ ข้อมูล Key Financial items ได้รับยืนยันและเพื่อเพิ่มความน่าเชื่อถือ และนำข้อมูลเหล่านี้มาวิเคราะห์ต่อยอดเพื่อค้นหาหา insight, Early warning sign และ Trend analysis 
2.2 ให้ AI ช่วยประเมิน Carbon Emission พร้อมกับคำนวณ Funding amount ที่ธนาคารสามารถเข้าไปปล่อยสินเชื่อ


Problem : สถานการณ์ปัจจุบัน ปัญหาและอุปสรรคที่พบในการทำงานปัจจุบัน แบ่งออกเป็น 2 ส่วน ดังนี้ 
1. Financial Statements
1.1 ข้อมูลในระบบของธนาคารนั้น ยังคงมีบาง Key ratio ที่สำคัญที่ยังไม่สมบรูณ์ เช่น Sales ในบางปียังขาดหายไป รวมทั้ง CAPEX ที่ไม่ได้มีการเก็บบันทึกข้อมูลไว้ ซึ่ง Key ratio ดังกล่าวเป็นข้อมูลสำคัญสำหรับการพัฒนา model forecasting เช่น financial statement forecasting เป็นต้น
1.2 ผลจากการประเมิน financial statement เป็นการประเมินรายลูกค้า ข้อมูลที่มีจึงมีขนาดใหญ่และมีรายละเอียดสำคัญ ทางทีมจึงใช้ระยะเวลานานในการวิเคราะห์ และคัดเลือกลูกค้าที่คาดว่าจะมี sign ที่แย่

2.Sustainable Development report
2.1 ใช้เวลาและทรัพยากรเป็นอย่างมากในการหาข้อมูล ESG เพื่อให้ในการประเมิน Finance emission ปัจจุบันนี้ 
2.2 ESG data source provider อย่าง Bloomberg ที่มีการรวบรวมข้อมูล ESG ของลูกค้า ยังอยู่ในระดับ Listed Company และยังให้ข้อมูล ESG ไม่ครบถ้วนเพียงพอ ในทุกมิติสำหรับการคำนวณ Finance emission ของธนาคาร

1. AI จะช่วยเก็บข้อมูล Key ratio ที่สำคัญจากไฟล์คำอธิบายงบการเงิน เพื่อลดเวลาในการเก็บข้อมูล และได้ข้อมูลที่ถูกต้อง พร้อมใช้งานในรูปแบบ format ที่ต้องการ
2. AI จะช่วยสรุปผลการประเมิน financial statement forecast เพื่อ trigger ลูกค้าหรืออุตสาหกรรมที่มีความเสี่ยงที่จะมีผลประกอบการแย่ ได้รวดเร็วและถูกต้อง

Project code: P11

Name : Social Media Fraud Signal Detection (ZX.)

Detail : Social Media Fraud Signal Detection: This project leverages cutting-edge technologies like Natural Language Processing (NLP), Graph Neural Networks (GNNs), and Computer Vision to proactively identify and mitigate fraud risks originating from social media platforms. By analysing public posts, profiles, and networks in real-time, this AI-powered solution detects bank logo misuse, fraudulent keywords, impersonation attempts, and scam networks, delivering actionable insights via interactive Power BI dashboards. With a focus on data privacy compliance, sentiment analysis, and context-aware AI models, this scalable and innovative system enhances customer protection, strengthens fraud prevention, and safeguards the banks reputation in the digital age.

Problem : 1. Increasing Fraud on Social Media
Social media platforms have become hotspots for fraudulent activities, including fake loan offers, impersonation of bank employees, and phishing scams targeting customers.
2. Customer Vulnerability
Many customers fall victim to scams due to misleading posts, aggressive marketing by fraudsters, or lack of awareness about online fraud tactics.
3. Impersonation and Reputation Risks
Fake profiles impersonating bank officials or support pages harm the banks credibility and lead to customer mistrust.
4. Lack of Real-Time Detection
Traditional fraud detection systems often fail to monitor social media platforms in real-time, allowing fraudsters to operate undetected for extended periods.
5. Complex Fraud Networks
Fraudsters often work in coordinated groups, making it challenging to detect patterns or connections between accounts without advanced tools.
6. Dynamic and Evolving Tactics
Fraudsters continuously adapt their methods, using new keywords, hashtags, and tactics to evade detection.
7. Reputation Damage
Negative sentiment arising from fraudulent activities linked to the bank can tarnish its image, leading to financial and customer trust losses.
8. Manual Monitoring Limitations
Human efforts to track and address fraud signals on social media are time-intensive, prone to errors, and lack scalability.


Solution : 
1. Fraud Signal Identification
1.1 Detect posts or advertisements claiming fraudulent services (e.g., "Guaranteed loan approval" or "Manipulate your credit score").
1.2 Monitor hashtags or keywords linked to scams (e.g., #QuickLoan, #FixCreditScore).
1.3 Analyse sentiment and intent of posts using Natural Language Processing (NLP).
2. Profile Behavior Analysis
Detect impersonation of bank officials or fake accounts pretending to represent the bank.
3. Network Analysis
3.1 Map relationships between accounts to identify networks of scammers or mule accounts.
3.2 Use graph analysis to find clusters of fraudulent activities or repeated patterns.
4. Customer Protection Alerts
Provide early warnings about potential risks associated with specific profiles or posts.


Outcome : 1. Proactive Fraud Prevention - Identifying fraud signals on social media before they impact customers or the bank.
2. Customer Safety - Protecting customers from falling victim to scams, enhancing trust in the bank.
3. Reputation Protection - Mitigating risks to the banks reputation by addressing fake profiles or negative propaganda.
4. Scalable Solution - The system can be scaled to monitor multiple platforms and languages, ensuring comprehensive coverage.

Name : AI-Driven Face Scan Verification with Metadata and Background Analysis (ZX.)

Detail : Develop an AI-powered verification system that combines facial recognition technology, metadata extraction, and background analysis to detect suspicious behaviours, such as coordinated account registrations or fraud attempts by groups operating as a gang. The system will analyse biometric features, metadata (e.g., date, IP address, geolocation), and image backgrounds to identify patterns of suspicious activities.

Problem : 1. Coordinated Fraud Rings
Organized fraud groups often register multiple accounts or perform suspicious activities in a coordinated manner, making it difficult to identify and disrupt their operations using traditional detection methods.
2. Ineffective Manual Reviews
Manual verification processes are time-consuming, prone to human error, and insufficient for detecting subtle patterns, such as shared metadata or recurring backgrounds across accounts.
3. Challenges in Liveness Detection
Existing facial recognition systems struggle with preventing spoofing attempts, such as using photos, videos, or 3D-printed masks to bypass biometric verification.
4. Overlooked Metadata and Context
Conventional verification systems often fail to analyze metadata (e.g., timestamps, IP addresses, geolocation) or background patterns, which can provide critical clues about fraud attempts.
5. Increasing Operational Workload
High volumes of account registrations and transactions increase the workload for fraud prevention teams, leading to escalating costs without significant improvements in detection accuracy.


Solution : 1. Face Scan Verification
- Use facial recognition algorithms to match face scans with official IDs or existing records.
- Include liveness detection to prevent spoofing through photos or videos.
2. Metadata and Background Analysis
- Extract and analyse metadata from the submitted images, including timestamps, IP addresses, and geolocation.
- Use image segmentation and background matching algorithms to identify similar or identical backgrounds across multiple submissions.
3. Pattern Recognition for Coordinated Activity
- Detect anomalies like multiple registrations or face scans completed from the same location or with similar backgrounds.
- Identify instances where suspicious individuals complete face scans with shared settings, suggesting coordinated actions.
4. Risk Scoring for Gang Activity
- Assign higher risk scores to activities that exhibit patterns of coordination (e.g., multiple accounts opened within a short time frame, shared backgrounds, or metadata anomalies).


Outcome : 1. Proactive Fraud Detection - Identifies coordinated activities and potential fraud rings before they cause significant damage.
2. Enhanced Security - Combines biometrics, metadata, and contextual analysis for a multi-layered approach to fraud prevention.
3. Operational Efficiency - Automates the detection of suspicious patterns, reducing manual investigation efforts.
4. Scalability - Adaptable to various scenarios, such as account openings, loan applications, and transaction authentications.

- Identify missing or tampered digital watermarks, seals, or QR codes to detect counterfeit documents.

Outcome : เสนอราคาโอนตีได้เหมาะสมกับราคามาตรฐานและราคาตลาด รวมถึงเสนอราคาขายทรัพย์NPA พร้อมทั้งMatchingกลุ่มผู้ซื้อทรัพย์ได้มากขึ้น ลดการตั้งสำรองNPAของธนาคารได้

Project code: P27

Name : Enhance Risk Scoring for Case Selection (CQ.)

Detail : Enhance risk score สำหรับการเลือก incoming policy lending case ขึ้นมาสอบทานประเด็นความเสี่ยง

Problem : เนื่องจากปัจจุบัน risk score ที่ใช้เพื่อเป็น guideline ในการเลือกเคสที่มีประเด็นความเสี่ยงเพื่อทำการสอบทานนั้นไม่สะท้อนต่อความเสี่ยงที่เกิดขึ้นจริง (ความเสี่ยงที่พบหลังสอบทานนั้น ไม่สอดคล้องกับความเสี่ยงที่ถูกประเมินไว้ก่อนทำการสอบทาน) ด้วยเหตุนี้ผู้สอบทานจึงต้องลงแรงลงเวลาไปกับการเลือกเคส (บางกรณีอาจใช้เวลาถึงครึ่งวัน หรือ บางกรณีอาจถึงขั้นที่ว่าเป็นการลงมือสอบทานเคสนั้นจนใกล้เสร็จสิ้นแล้วเสียด้วยซ้ำ) เพื่อให้สามารถหยิบเคสที่มีประเด็นความเสี่ยงขึ้นมาสอบทาน

นอกเหนือจากประเด็น risk score ไม่สะท้อนฯ ซึ่งก่อให้เกิดขั้นตอนการเลือกเคสที่ไม่ efficient ก็ยังเชื่อมโยงไปถึง concern ในมุมของ effectiveness ในส่วนของความแม่นยำ (ลง resource ไปกับการสอบทานเคสที่ไม่มีประเด็นเยอะเกินไปหรือไม่) และความครอบคลุม (เคสที่มีประเด็นความเสี่ยงที่ไม่ถูกหยิบมาสอบทานนั้นหลุดไปเยอะหรือไม่) ด้วยเช่นกัน

Solution : ช่วยในการคำนวณ risk score ให้สอดคล้องต่อความเสี่ยงที่เกิดขึ้นจริงมากยิ่งขึ้น เพื่อให้สามารถเลือกเคสมาตรวจได้ effective มากขึ้น (เพิ่มความแม่นยำ precision และความครอบคลุม recall) และ efficient มากขึ้น (ideally ผู้สอบทานสามารถเลือก case ที่มี risk score สูงสุดได้เลยโดยไม่ต้อง manually เลือก case เอง)

Outcome : ในมุม effectiveness จะถูกวัดในมุม precision และ recall
- Boost precision 2x from 4% to 8% (at least)
- existing recall 64% -> expected recall >= 85% (at least)

ในมุม efficiency จะถูกวัดในมุม time spent on case selection
- expect saving time 4 hours ต่อคนต่อรอบ  (2 รอบต่อสัปดาห์)

2. ใช้ AI เพื่อตรวจสอบข้อความที่ OCR อ่านมาได้ และแยกเป็นข้อมูลที่ธนาคารจำเป็นต้องใช้ และนำไปตรวจสอบเทียบกับฐานข้อมูล ว่าหมายนั้นๆ ตรงตามกับอำนาจหน้าที่ในการขอข้อมูล หรือสามารถสั่งการ อายัด ปลดอายัด ได้หรือไม่
3. ใช้ RPA นำข้อมูลที่ได้ ไปใช้ในการดึงข้อมูลต่างๆ หรือดำเนินการ อายัด/ปลดอายัด ตามที่หมายคำสั่งแจ้งให้ดำเนินการ

Outcome : โครงการนี้จะสามารถทำให้ธนาคารสามารถตอบสนอง ต่อคำสั่งหน่วยงานราชการได้อย่างมีประสิทธิภาพยิ่งขึ้น และสามารถรองรับปริมาณงานที่เพิ่มขึ้นได้โดยไม่จำเป็นต้องเพิ่มผู้ปฏิบัติงาน

Project code: P31

Name : Data Right ,Trade Bright (MR.)

Detail : สร้าง AI ตรวจสอบความถูกต้องของการบันทึกธุรกรรมของพนักงานบนระบบงาน ที่มาจากเสียงสนทนาการตกลงทำธุรกรรมของลูกค้า/คู่ค้า ธุรกิจห้องค้าตลาดเงินและตลาดทุน


Problem : 1. การทำงานของ Sales ปัจจุบันยังมีความเสี่ยงเรื่องการบันทึกข้อมูลธุรกรรมที่มาจากลูกค้า/คู่ค้าลงบนระบบผิดพลาด เนื่องจากเป็นโทรสนทนากับลูกค้า/คู่ค้า และจดรายละเอียดลงบนกระดาษเพื่อนำไปบันทึกข้อมูลธุรกรรมบนระบบงาน
2. Sales ต้องใช้เวลาในแต่ละวันเพื่อมา reconcile โดยการ cross-check รายละเอียดธุรกรรมที่เกิดขึ้น ทุกสิ้นวัน

Solution : 1. ช่วยให้การบันทึกข้อมูลธุรกรรมของ Sales ลงในระบบ มีความแม่นยำถูกต้องมากขึ้น โดยการ cross-check กับ AI ที่แปลงจากเสียงการสนทนาเป็นตัวหนังสือ
2. ช่วยลดกระบวนการที่ Sales ต้องมา reconcile รายละเอียดธุรกรรมทุกสิ้นวัน โดย AI สามารถ cross-check ข้อมูลได้แทน

Outcome : 9.1) ลดจำนวนการบันทึกธุรกรรมผิดพลาดของ Sales
9.2) ลดระยะเวลาการทำงานในการ reconcile รายละเอียดธุรกรรมในแต่ละวัน โดย Sales สามารถไปทำงานอย่างอื่น หรือรับ Deal ได้มากขึ้น

Project code: P32

Name :  WBG AI Risk Data Analytics (ZZ.)

Detail : ใช้ AI และ tool ในการแสดงผลและตอบคำถาม งาน Risk Data ของสายงาน regional ERM-WBG

Problem : ปัจจุบัน ผู้บริหารและพนักงานมีความต้องการเข้าถึงข้อมูลอย่างรวดเร็วและทันที ส่งผลให้บางครั้งไม่สามารถตอบคำถามได้อย่างทันท่วงที ซึ่งต้องเสียเวลาในการเตรียมข้อมูลในส่วนนี้เป็นอย่างมาก จึงมีความต้องการลดภาระงานในส่วนดังกล่าวลง.

Solution : AI ช่วยสรุปผล ตอบคำถาม แสดงภาพ กราฟ ตามข้อมูลที่มีอยู่ใน KBank

Outcome : - ลด working-hours ของฝ่ายงาน
- เพิ่มความครบถ้วน และ การแสดงผลในแง่มุมที่เพิ่มขึ้น
- ผู้บริหาร ได้รับข้อมูล ทันที
